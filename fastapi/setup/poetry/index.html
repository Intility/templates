Without PyCharm (poetry)

Virtual environmentâ€‹

There are two options for creating your virtual environment, either manually through poetry or PyCharm.

info

This section will cover how to use poetry. Head over to the previous page if you want to use PyCharm instead.

Creating a virtual environmentâ€‹

info

Its easier to deal with poetry when the virtual environment is placed in your working directory. You can configure poetry to do this by issuing this command:

poetry config virtualenvs.in-project true

First create a virtual environment and install dependencies:

poetry update

The update command both updates the poetry.lock file and creates the virtual environment.

Activate your new virtual environment:

poetry shell

Migrationsâ€‹

You can skip this section if you didn't select sqlmodel and have a database.

In order to configure the database tables to reflect your models, you must run migrations. In this project, we use alembic to manage our migrations. Running migrations is done through the terminal:

alembic upgrade head

Running the serverâ€‹

With your virtual environment activated and docker-compose running, we can run our server.

poetry run uvicorn app.main:app --host localhost --port 8000 --reload
